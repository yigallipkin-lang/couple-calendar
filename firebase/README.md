# Couple Calendar - Firebase Configuration & Security

## Overview

This directory contains all Firebase-related configuration, security rules, and documentation for the Couple Calendar application.

## Quick Start

### 1. Setup

```bash
# Install Firebase CLI
npm install -g firebase-tools

# Login to Firebase
firebase login

# Verify project
firebase projects:list
```

### 2. Deploy Rules

```bash
# From project root
firebase deploy --only firestore:rules
```

### 3. Test Rules

```bash
# Start emulator
firebase emulators:start

# In another terminal
firebase test
```

## Collection Schema

### Users
```
/users/{userId}
├── email: string
├── displayName: string
├── color: string (hex)
├── partnerId: string | null
├── createdAt: timestamp
└── photoURL: string (optional)
```

### Couples
```
/couples/{coupleId}
├── partner1Id: string
├── partner2Id: string
├── pairedAt: timestamp
└── settings: object
    ├── theme: string
    ├── language: string
    └── notificationsEnabled: boolean
```

### Events
```
/events/{coupleId}/items/{eventId}
├── title: string
├── startTime: timestamp | null
├── endTime: timestamp | null
├── allDay: boolean
├── color: string
├── notes: string (optional)
├── location: string (optional)
├── reminder: string
├── checklist: array (optional)
├── createdBy: string
├── createdAt: timestamp
└── updatedAt: timestamp
```

### Todo Lists
```
/todoLists/{coupleId}/items/{itemId}
├── text: string
├── done: boolean
├── createdAt: timestamp
└── updatedAt: timestamp
```

### Pairing Codes
```
/pairing_codes/{code}
├── code: string
├── generatorId: string
├── createdAt: timestamp
├── expiresAt: timestamp
├── used: boolean
├── acceptedBy: string (optional)
└── acceptedAt: timestamp (optional)
```

## Security Model

### Principles

1. **Authentication Required** - All access requires login
2. **Couple Isolation** - Users only access data for their couple
3. **Membership Verification** - Every operation verifies couple membership
4. **Least Privilege** - Users can only modify their own data
5. **Cloud Functions Managed** - Sensitive operations handled by Cloud Functions

### Access Rules

| Collection | Owner | Partner | Others | Unauthenticated |
|-----------|-------|---------|--------|-----------------|
| Own User Doc | RW | - | - | - |
| Couple Doc | RW | RW | - | - |
| Events | RW | RW | - | - |
| Todo Lists | RW | RW | - | - |
| Own Notifications | R | - | - | - |
| Pairing Codes | R | R | R | - |

Legend:
- `RW` = Read & Write
- `R` = Read only
- `-` = Denied
- Cloud Functions have admin access and bypass all rules

## Monitoring & Alerts

### Key Metrics to Monitor

1. **Permission Denied Errors**
   - Watch for spikes in 403 errors
   - May indicate rule misconfiguration
   - Check Cloud Logging: `protoPayload.status.code=403`

2. **Real-time Listener Health**
   - Monitor snapshot updates
   - Check for delayed updates
   - Verify network connectivity

3. **Cloud Function Execution**
   - Monitor function logs
   - Check for validation failures
   - Alert on abnormal execution times

### Firestore Quotas

Monitor usage in Firebase Console:
- **Reads per day** - Each query counts as 1 read
- **Writes per day** - Each write counts as 1 write
- **Deletes per day** - Each delete counts as 1 delete
- **Storage size** - Monitor data growth

Free tier limits:
- 50,000 reads/day
- 20,000 writes/day
- 20,000 deletes/day

## Troubleshooting

### "Permission denied" Error

```typescript
// Check authentication
if (!auth.currentUser) {
  // User not logged in
  redirect to login
}

// Check couple membership
const userRef = doc(db, 'users', auth.currentUser.uid);
const userData = await getDoc(userRef);
const partnerId = userData.data().partnerId;

if (!partnerId) {
  // User not paired yet
  redirect to pairing flow
}

// Check couple document exists
const coupleId = [auth.currentUser.uid, partnerId].sort().join('_');
const coupleRef = doc(db, 'couples', coupleId);
const coupleData = await getDoc(coupleRef);

if (!coupleData.exists()) {
  // Couple document missing - should not happen
  console.error('Couple document not found');
}
```

### Real-time Listener Not Updating

```typescript
// Verify listener is active
const unsubscribe = onSnapshot(
  query,
  (snapshot) => {
    console.log('Listener triggered', snapshot.docs.length);
  },
  (error) => {
    console.error('Listener error:', error);
    // error.code === 'permission-denied' = security rule blocked
  }
);

// Call unsubscribe when component unmounts
return () => unsubscribe();
```
